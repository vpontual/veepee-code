# v0.2.0 Plan: Compressed Knowledge State

## Problem
Every Ollama API call sends the full conversation history (system prompt + all messages). A 20-message conversation means ~8,000+ tokens sent with EVERY request. This is slow, wasteful, and hits context limits fast.

## Solution
Replace full history with a **compressed knowledge state file** (~200 tokens) that gets updated every turn. The model reads this instead of scrolling through 50 messages.

## Architecture

### Knowledge State File
Location: `~/.veepee-code/sessions/{session-id}-state.md`

Format (AI-optimized, not human-readable):
```
PROJECT: veepee-code | TypeScript | CLI coding tool
CWD: ~/Development/veepee-code
USER_INTENT: fixing auth module
DECISIONS: [use JWT not sessions, store in httpOnly cookie]
FILES_READ: [src/auth.ts, src/middleware.ts, package.json]
FILES_MODIFIED: [src/auth.ts:added validateToken()]
CURRENT_TASK: write tests for validateToken
FACTS: [uses express 4.x, jest for testing, no existing auth tests]
ERRORS: [TypeError in auth.ts:42 — fixed]
OPEN_QUESTIONS: [should tokens expire after 24h or 7d?]
TURN: 8
```

### Message Flow Per API Call
**Before (v0.1):**
```
System prompt (~3000 tok)
+ Message 1: user "hi"
+ Message 2: assistant "Hello!"
+ Message 3: user "read auth.ts"
+ Message 4: assistant [tool_call: read_file]
+ Message 5: tool result (500 lines)
+ ... 15 more messages ...
+ Message 20: user "now write the tests"
= ~8,000+ tokens per call
```

**After (v0.2):**
```
System prompt (~2000 tok, slimmed)
+ Knowledge state (~200 tok)
+ Last 3 messages (~500 tok)
+ New message
= ~2,800 tokens per call
```

### Implementation Steps

#### Step 1: Create KnowledgeState class (`src/knowledge.ts`)
- `KnowledgeState` class with structured fields
- `update(messages, toolResults)` — extracts key facts from recent messages
- `serialize()` — outputs the compact format
- `deserialize(text)` — reads it back
- `toSystemPromptBlock()` — formatted for injection into system prompt

#### Step 2: Update ContextManager (`src/context.ts`)
- After each agent turn, call `knowledgeState.update()`
- Build messages array as: system prompt + knowledge state + last 3 messages
- Remove the full history from the messages array
- Keep full history in memory for `/save` sessions (but don't send to API)

#### Step 3: Update system prompt
- Add instruction: "Your knowledge state is shown below. It contains everything important from our conversation. Only the last 3 messages are shown. If you need older details, check the knowledge state."
- Slim the system prompt — remove verbose examples, tighten rules

#### Step 4: Knowledge state auto-update
- After each assistant response, extract:
  - Files read/modified (from tool calls)
  - Decisions made (from assistant text — look for "I'll use", "let's go with", "decided")
  - Errors encountered (from tool results with errors)
  - Current task (from latest user message intent)
- Model can also explicitly update via a `update_memory` tool call

#### Step 5: Update /save and /resume
- Save knowledge state alongside session
- Resume restores knowledge state, not full message replay

#### Step 6: Add `update_memory` tool
- New tool the model can call: `update_memory(key, value)`
- Updates the knowledge state with explicit facts
- System prompt tells model: "When you learn something important, call update_memory"
